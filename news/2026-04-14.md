## 📖 用語解説

| 用語 | 解説 |
|------|------|
| **トークン** | AI モデルが文章を処理する際の最小単位。単語や文字のかたまりで、日本語では1文字〜数文字が1トークンに相当。コンテキストウィンドウの上限はトークン数で管理される。 |
| **コンテキストウィンドウ** | AI が一度に「覚えていられる」情報量の上限。1M トークンは日本語でおよそ100万文字（文庫本数百冊分）に相当し、長大なコードベースや文書を丸ごと扱える。 |
| **拡張思考（Extended Thinking）** | 回答を返す前に Claude が内部で多段階の推論ステップを実行する機能。複雑な問題や数学・コーディングタスクで精度が向上する。 |
| **エージェント / AIエージェント** | 人間の指示なしに自律的に行動・判断・ツール利用を繰り返して目標を達成する AI システム。単発の質問応答とは異なり、連続的なタスク実行が特徴。 |
| **サンドボックス** | プログラムを隔離された安全な実行環境で動かす仕組み。エージェントが誤った操作をしても本番環境や他のシステムに影響が出ないようにする。 |
| **SSE（Server-Sent Events）** | サーバーからクライアントへリアルタイムにデータを送り続けるWeb技術。AI の回答を「ストリーミング」で少しずつ表示する際に使われる。 |
| **CLI（Command Line Interface）** | マウスを使わずキーボードでコマンドを入力して操作するインターフェース。`ant` や `Claude Code` はターミナルから使う CLI ツール。 |
| **MCP（Model Context Protocol）** | Anthropic が策定した、AI モデルと外部ツール・データソースを標準的に接続するためのオープンプロトコル。USB のように「差し込むだけ」で AI がツールを使えるようにする仕組み。 |
| **Transformer** | 現代の大規模言語モデル（LLM）の基盤となるニューラルネットワーク構造。GPT・Claude・Gemini などほぼすべての主要 AI モデルがこのアーキテクチャを採用している。 |
| **オープンソース** | ソースコード（設計図）を誰でも無償で閲覧・利用・改変・再配布できる形で公開すること。研究や商用利用が自由に行えるため普及が速い。 |
| **ペネトレーションテスト（pentest）** | システムのセキュリティを検証するために、攻撃者の視点で意図的に侵入を試みるテスト手法。「侵入テスト」とも呼ばれ、脆弱性の発見に使われる。 |
| **Coordinator Mode** | 複数のサブエージェントを並行して動かし、それぞれの作業を調整・統合する役割を担うモード。大規模タスクを分割して効率的に処理できる。 |
| **DB スキーマ** | データベースの構造定義（テーブル名・カラム名・データ型など）をまとめた設計図。AI がデータベースを操作する際に参照する情報で、サイズが大きくなりやすい。 |
| **量子コンピューティング** | 量子力学の原理（重ね合わせ・もつれ）を利用した次世代コンピューター技術。特定の計算で従来型コンピューターを圧倒的に上回る速度が期待されており、AI との融合研究が進む。 |
