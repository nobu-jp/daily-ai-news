## 💡 Claude Code / MCP 関連

- **Claude Code v2.1.92リリース（4月4日）**: 主な変更点:
  - `forceRemoteSettingsRefresh` ポリシー設定（マネージドリモート設定用）
  - Amazon Bedrockセットアップウィザードをインタラクティブに実行可能
  - `/cost` コマンドにモデル別・キャッシュヒット別の内訳表示を追加（サブスクリプションユーザー）
  - `/release-notes` でバージョンピッカーが使えるように
  - Writeツールのdiff計算を60%高速化
  - `/tag` と `/vim` コマンドを削除
  - 情報源: [GitHub Releases - claude-code](https://github.com/anthropics/claude-code/releases)

- **Claude Code v2.1.91（4月2日）— MCPツール結果の永続化**: `_meta["anthropic/maxResultSizeChars"]` アノテーションでMCPツール結果の保持サイズを上書き可能に（最大500K文字）。`disableSkillShellExecution` 設定、マルチラインプロンプトのディープリンク対応も追加。
  - 情報源: [GitHub Releases - claude-code](https://github.com/anthropics/claude-code/releases)

- **Claude Code v2.1.90（4月1日）— `/powerup` インタラクティブレッスン追加**: アニメーションデモつきのインタラクティブ学習コンテンツが利用可能に。MCPツールスキーマのJSON.stringifyを排除しパフォーマンス向上。SSEトランスポートの大規模フレーム処理を二乗→線形に改善。
  - 情報源: [GitHub Releases - claude-code](https://github.com/anthropics/claude-code/releases)

- **MCP関連バグ修正（v2.1.91〜v2.1.92）**:
  - `deniedMcpServers` 設定がclaude.ai MCPサーバーをブロックしない問題を修正
  - MCPステップアップ認可でリフレッシュトークンが存在する場合に失敗する問題を修正（403 `insufficient_scope` 時の再認可フロー）
  - マルチモニター環境でのcomputer-useツールの`switch_display`エラーを修正
  - 情報源: [GitHub Releases - claude-code](https://github.com/anthropics/claude-code/releases) / [Apiyi.com Blog](https://help.apiyi.com/en/claude-code-v2-1-92-mcp-persistence-powerup-tutorial-en.html)
